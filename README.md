# Описание базы данных
## Users
+ user_id — первичный ключ (PK), уникальный идентификатор пользователя.
+ email — адрес электронной почты пользователя, должен быть уникальным.
+ login — логин пользователя для входа, должен быть уникальным.
+ name — полное имя пользователя.
+ birthday — дата рождения пользователя.

## Friendship
> [!NOTE]
> (user_id, friend_id) — составной первичный ключ (PK). Уникальная пара определяет конкретную связь между двумя пользователями. Отдельно user_id или friend_id не гарантируют уникальность записи.
+ user_id — идентификатор пользователя, который инициировал связь (или указан первым в паре), внешний ключ (FK) ссылается на users.user_id.
+ friend_id — идентификатор пользователя, к которому обращена связь, внешний ключ (FK) ссылается на users.user_id.
+ friendship_status — статус дружбы (например, PENDING — запрос отправлен, CONFIRMED — друзья).

## Films
+ film_id — первичный ключ (PK), уникальный идентификатор фильма.
+ name — название фильма.
+ description — описание или сюжет фильма.
+ release_date — дата выхода фильма.
+ duration — длительность фильма в минутах.
+ mpa_rating — рейтинг Ассоциации кинокомпаний (MPA), определяющий возрастное ограничение (G, PG, PG-13, R, NC-17).

## Genres
+ genre_id — первичный ключ (PK), уникальный идентификатор жанра.
+ name — название жанра, должно быть уникальным (UNIQUE), чтобы избежать дублирования (например, двух записей «Комедия»).

## Films_genre
> [!NOTE]
>(film_id, genre_id) — составной первичный ключ (PK). Гарантирует, что один и тот же жанр нельзя привязать к одному фильму дважды. Уникальность обеспечивается только парой значений.

+ film_id — идентификатор фильма, внешний ключ (FK) ссылается на films.film_id.
+ genre_id — идентификатор жанра, внешний ключ (FK) ссылается на genres.genre_id.

## Liked_films
> [!NOTE]
>(user_id, film_id) — составной первичный ключ (PK). Обеспечивает правило: один пользователь может поставить лайк одному фильму только один раз.

+ user_id — идентификатор пользователя, поставившего лайк, внешний ключ (FK) ссылается на users.user_id.
+ film_id — идентификатор лайкнутого фильма, внешний ключ (FK) ссылается на films.film_id.
